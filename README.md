学习过程中发现，如果在外部想要添加一个线程池用于执行MessageCallback回调获取的任务，如果执行的任务中调用了发送函数例如conn->send(msg)，由于send接收的是一个引用，如果**msg被销毁而线程池的任务还没有得到执行**，将会发生未定义错误。最简单的例子就是std::string如果是长字符串，内部会变成一个指针，当生命周期结束这个指针被释放了后续执行send任务的时候string将发送乱码，但短字符串有时不会被影响，因为数据直接存储在 std::string 对象内部的栈缓冲区里，即使外部 std::string 销毁，指针指向的内存仍然在栈上（可能还没被覆盖），所以看起来“没问题”或偶尔正常。 
所以提升项目为C++14标准，使用C++14的**初始化捕获特性（Init-capture）**修改send函数，将对象移动到lambada内部，让数据所有权跟随任务（lambda/functor）走，延长变量的生命周期至任务结束。
